# Codenix

**Codenix** es una plataforma web para practicar programación competitiva y algoritmos mediante problemas, envíos de soluciones y seguimiento de progreso.

El objetivo del proyecto es construir una experiencia moderna, ordenada y fácil de usar para estudiantes, desarrolladores y personas que quieran mejorar resolviendo problemas de programación.

## ¿Qué puedes hacer aquí?

En Codenix se busca ofrecer una plataforma donde los usuarios puedan:

* Explorar problemas de programación.
* Escribir y enviar soluciones.
* Ejecutar código en lenguajes soportados.
* Revisar el resultado de sus envíos.
* Seguir su progreso dentro de la plataforma.
* Practicar con una interfaz clara e inspirada en plataformas modernas de coding.

La primera versión del proyecto prioriza una base sólida: interfaz, estructura del sistema, flujo de práctica y documentación técnica clara.

## Stack tecnológico

El proyecto se está construyendo con un enfoque web moderno y modular.

### Frontend

* React
* TypeScript
* Vite
* Tailwind CSS
* Mock data para la primera etapa de desarrollo

### Backend y juez en línea

El backend y el sistema de juez en línea forman parte de la arquitectura planificada del proyecto. La documentación técnica detallada se mantendrá dentro de la carpeta `docs/`.

Lenguajes objetivo para ejecución de soluciones en la primera versión:

* Python
* Java
* C

> Nota: esta sección se actualizará conforme se consoliden las decisiones técnicas del backend y del juez.

## Instalación y puesta en marcha

### Requisitos

Antes de empezar, asegúrate de tener instalado:

* Node.js
* npm
* Git

### Clonar el repositorio

```bash
git clone https://github.com/<usuario>/<repositorio>.git
cd <repositorio>
```

### Instalar dependencias

```bash
npm install
```

### Ejecutar el entorno de desarrollo

```bash
npm run dev
```

Luego abre la URL local que indique la terminal, normalmente:

```bash
http://localhost:5173
```

## Uso básico

En la primera etapa, el proyecto puede ejecutarse con datos simulados para construir y validar la experiencia del frontend.

Flujo esperado:

1. Abrir la aplicación en el navegador.
2. Explorar la lista de problemas disponibles.
3. Entrar al detalle de un problema.
4. Revisar el enunciado y la interfaz de solución.
5. Simular el flujo de envío y revisión de resultados.

A medida que avance el backend y el juez en línea, este flujo se conectará con datos reales y ejecución controlada de código.

## Contribución

Las contribuciones son bienvenidas, especialmente en documentación, interfaz, estructura del proyecto y mejoras de experiencia de usuario.

Antes de contribuir, revisa la documentación del proyecto y trabaja desde una rama separada. Los cambios deben enviarse mediante pull request para mantener un flujo ordenado.

La guía completa de contribución estará disponible en la carpeta `docs/`.
